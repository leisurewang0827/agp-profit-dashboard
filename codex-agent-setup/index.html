Codex 에이전트 시작 준비물

Codex는 단순 채팅 도구가 아니라 코드 수정, 브라우저 조작, 데이터베이스 조회, 업무 시스템 확인, 리포트 자동화까지 맡길 수 있는 작업 에이전트입니다. 아래 항목은 처음 세팅할 때 필요한 프로그램과 계정 권한을 정리한 체크리스트입니다.

가볍게 시작 ChatGPT/Codex 접속, GitHub 연결, 기본 저장소 접근 권한만 있으면 시작할 수 있습니다.
로컬 작업 Git, Node.js, Python, Codex CLI를 설치하면 내 컴퓨터 파일을 다루는 작업까지 가능합니다.
업무 자동화 Playwright, Browser Use, Supabase, Slack, Notion, 광고/쇼핑몰/회계 시스템 권한과 필요한 API 키가 필요합니다.

1. 기본 프로그램

Google Chrome

로그인된 업무 사이트를 확인하고 브라우저 자동화를 실행할 때 필요합니다.

코드 에디터

필수는 아닙니다. 필요하면 VS Code, Cursor, Zed 등 익숙한 에디터를 선택해서 쓰면 됩니다.

Git

GitHub 저장소를 내려받고 변경사항을 관리하기 위해 필요합니다.

Node.js + npm

Codex CLI, Playwright, 브라우저 자동화 도구를 설치하고 실행하는 기반입니다.

Python 3

CSV/Excel 처리, 데이터 분석, 자동화 스크립트 실행에 자주 사용됩니다.

Homebrew

macOS에서 개발 도구와 CLI 프로그램을 설치·업데이트할 때 편합니다.

2. Codex 관련 설치

  • ChatGPT/Codex 계정: Codex web 또는 ChatGPT에서 Codex를 사용할 수 있는 계정이 필요합니다.
  • Codex CLI: 로컬 터미널에서 Codex를 실행하려면 npm i -g @openai/codex로 설치합니다.
  • GitHub 연결: 저장소 코드 수정, 브랜치 생성, 커밋, PR 작업을 맡기려면 GitHub 권한이 필요합니다.
  • Codex 플러그인/커넥터: Slack, Notion, GitHub, Browser Use, Chrome, Documents, Spreadsheets, Presentations 등 필요한 기능을 켭니다.

3. 브라우저 자동화 준비

Playwright

웹사이트를 열고, 클릭하고, 데이터를 읽고, 화면을 검증하는 자동화 도구입니다.

Browser Use (iab)

Codex 안에서 브라우저 상태를 읽고 조작하는 인앱 브라우저 방식입니다.

Codex Chrome Extension

로그인된 Chrome 탭을 Codex가 안전하게 읽고 조작할 수 있게 해주는 런타임입니다.

Chrome 로그인 프로필

Instagram, Google, Naver, Meta처럼 로그인 상태가 중요한 업무는 저장된 Chrome 프로필을 씁니다.

로그인 필요한 사이트는 매번 새 브라우저를 띄우는 방식보다, 본인이 직접 로그인한 Chrome 세션을 Codex가 연결해서 쓰는 방식이 안정적입니다.

4. 데이터베이스와 업무 시스템 권한

Supabase

대시보드, 리포트, 자동화 결과를 저장하고 검증하는 데이터베이스 역할을 합니다.

Slack

자동화 결과, 실패 알림, 일일 리포트를 팀 채널로 보내는 데 필요합니다.

Notion

업무 문서, 운영 지식, 계정 정보 위치, 회의/작업 기록을 참조할 때 사용합니다.

경리나라

월마감, 손익표, 카드·은행 거래 확인, 회계 분류 작업에 필요합니다.

Meta Ads Manager

캠페인, 광고세트, 광고, 예산 구조 분석과 승인된 광고 변경에 필요합니다.

Google Ads / GA4

광고비, 매출, 전환, 유입 데이터를 확인하고 수익 분석에 연결합니다.

Naver SmartStore / SearchAd

네이버 매출, 정산, 검색광고 비용, 상품별 성과를 수집할 때 필요합니다.

Imweb / EZ Admin / KakaoTalk

자사몰 주문, 출고·재고, 발주, 거래처 커뮤니케이션 자동화에 사용합니다.

5. 발급해야 하는 API/키 리스트

모든 API가 처음부터 필요한 것은 아닙니다. 먼저 읽기 전용 리포트에 필요한 것부터 연결하고, 광고 변경·DB 쓰기·외부 메시지 발송 권한은 운영 방식이 정해진 뒤 추가하는 편이 안전합니다.

구분 발급/준비할 것 쓰는 곳 처음부터 필요?
OpenAI ChatGPT/Codex 계정, 필요 시 OpenAI API key Codex 사용, 별도 API 기반 자동화 필수
GitHub GitHub 로그인, 저장소 권한, 필요 시 Personal Access Token 코드 저장소 접근, 커밋, 푸시, PR 필수
Supabase Project URL, anon/service role key, database URL, access token 리포트 저장, 대시보드 데이터, 검증 쿼리 필수
Slack Slack connector 권한 또는 bot token 자동화 결과를 채널에 발송 필수
Notion Notion connector 권한 또는 integration token 운영 문서, 계정 위치, 업무 지식 참조 선택
Meta Ads Meta app, Marketing API access token, ad account ID, business ID 캠페인/광고세트/광고 구조 분석, 승인된 광고 작업 필수
Google Ads Developer token, OAuth client, refresh token, customer ID 광고비 수집, 채널별 수익 분석 선택
GA4 Property ID, OAuth 또는 service account 권한 매출, 전환, 유입, 광고성과 검증 선택
Naver SearchAd API key, secret key, customer ID 네이버 검색광고 비용/성과 수집 선택
Naver Commerce Commerce API application ID/secret, 판매자 권한 스마트스토어 주문/상품/정산 데이터 확인 선택
Imweb API key, API secret, 관리자 권한 자사몰 주문/매출 수집 선택
경리나라 / EZ Admin / KakaoTalk 대부분 API보다 로그인 권한과 화면/파일 접근이 중요 월마감, 재고/발주, 거래처 커뮤니케이션 확인 업무별
비밀번호, 토큰, API key는 문서나 Slack에 그대로 적지 말고, 환경변수·비밀 저장소·커넥터 권한으로 관리해야 합니다.

6. MCP/커넥터 쉽게 이해하기

MCP와 커넥터는 Codex가 외부 도구를 “직접 만질 수 있게 해주는 연결 통로”입니다. 사람으로 치면 로그인된 업무 앱에 들어갈 수 있는 손잡이와 권한표라고 보면 됩니다.

연결 쉽게 말하면 주요 역할 주의점
Browser Use (iab) Codex 안의 브라우저 페이지 열기, 화면 확인, 클릭, 간단한 웹 검증 로그인 세션이 별도일 수 있습니다.
Playwright MCP 브라우저 자동 조종 장치 반복 수집, 화면 테스트, 로그인된 Chrome 프로필 활용 로그인 작업은 저장된 안전한 프로필을 써야 합니다.
Chrome Extension 내 Chrome 탭과 Codex를 연결 Instagram, ChatGPT, Naver, Meta처럼 로그인된 페이지 확인 사용자가 로그인한 세션을 존중해야 합니다.
Supabase MCP DB 조회/검증 연결 대시보드 데이터 확인, 저장 결과 검증 쓰기 작업은 승인 절차를 두는 게 안전합니다.
GitHub connector 저장소 작업 연결 파일 확인, 브랜치, 커밋, PR 작업 민감 파일이 커밋되지 않게 확인해야 합니다.
Slack connector 팀 채널 알림 연결 리포트 발송, 실패 알림, 업무 요약 공유 외부 발송이므로 자동 전송 범위를 정해야 합니다.
Notion connector 업무 지식창고 연결 운영 문서 검색, 회의/업무 기록 참조 비공개 문서 접근 권한을 제한해야 합니다.

7. Meta 광고 시스템 구조 이해

  • Campaign: 광고 목적과 큰 예산 단위입니다. 캠페인 예산 방식이면 여기서 예산이 관리됩니다.
  • Ad Set: 타깃, 노출 위치, 일정, 일부 예산이 관리되는 단위입니다. ABO 구조에서는 광고세트 예산이 중요합니다.
  • Ad: 실제 소재, 문구, 랜딩 페이지가 붙는 단위입니다.
  • ASC: 성과가 검증된 소재를 더 넓게 확장할 때 쓰는 자동화 캠페인 구조입니다.
  • 운영 원칙: 분석은 자동화할 수 있지만, 예산 변경·광고 생성·ON/OFF 같은 실제 쓰기 작업은 명확한 승인 후 실행해야 합니다.

8. 현재 활용 중인 자동화 범위

실제 운영에서는 아래 같은 업무를 Codex 에이전트가 정해진 시간이나 요청에 맞춰 수행할 수 있습니다.

수익 대시보드 광고 성과 분석 Meta 세팅 검토 재고 리포트 발주 관리 거래처 카톡 요약 네이버 메일 확인 노무 지원사업 모니터링 보안 점검 Git 업데이트

9. 추천 세팅 순서

  1. Chrome, Git, Node.js, Python 3, Homebrew를 먼저 설치합니다.
  2. Codex 계정에 로그인하고 GitHub 저장소 접근 권한을 연결합니다.
  3. Codex CLI, Playwright, Supabase CLI를 설치해 로컬 작업과 브라우저 자동화를 준비합니다.
  4. GitHub, Slack, Supabase, Browser Use, Chrome, Notion 커넥터를 필요한 순서대로 연결합니다.
  5. Meta, Google, Naver, Imweb 등 API key가 필요한 서비스는 읽기 권한부터 발급합니다.
  6. EZ Admin, 경리나라, KakaoTalk처럼 API보다 로그인 화면이 중요한 서비스는 본인이 직접 로그인해 세션을 준비합니다.
  7. 처음에는 읽기 전용 리포트 자동화부터 시작하고, 광고 변경·DB 쓰기·외부 메시지 발송은 승인 절차를 둡니다.
이 문서는 개인 로컬 경로, 비밀번호, 토큰, 내부 저장소 설치 명령을 포함하지 않는 공유용 요약본입니다.

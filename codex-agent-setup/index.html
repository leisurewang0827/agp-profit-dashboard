Codex 에이전트 시작 준비물

Codex는 단순 채팅 도구가 아니라 코드 수정, 브라우저 조작, 데이터베이스 조회, 업무 시스템 확인, 리포트 자동화까지 맡길 수 있는 작업 에이전트입니다. 아래 항목은 처음 세팅할 때 필요한 프로그램과 계정 권한을 정리한 체크리스트입니다.

가볍게 시작 ChatGPT/Codex 접속, GitHub 연결, 기본 저장소 접근 권한만 있으면 시작할 수 있습니다.
로컬 작업 Git, Node.js, Python, Codex CLI를 설치하면 내 컴퓨터 파일을 다루는 작업까지 가능합니다.
업무 자동화 Playwright, Browser Use, Supabase, Slack, Notion, 광고/쇼핑몰/회계 시스템 권한이 필요합니다.

1. 기본 프로그램

Google Chrome

로그인된 업무 사이트를 확인하고 브라우저 자동화를 실행할 때 필요합니다.

코드 에디터

필수는 아닙니다. 필요하면 VS Code, Cursor, Zed 등 익숙한 에디터를 선택해서 쓰면 됩니다.

Git

GitHub 저장소를 내려받고 변경사항을 관리하기 위해 필요합니다.

Node.js + npm

Codex CLI, Playwright, 브라우저 자동화 도구를 설치하고 실행하는 기반입니다.

Python 3

CSV/Excel 처리, 데이터 분석, 자동화 스크립트 실행에 자주 사용됩니다.

Homebrew

macOS에서 개발 도구와 CLI 프로그램을 설치·업데이트할 때 편합니다.

2. Codex 관련 설치

  • ChatGPT/Codex 계정: Codex web 또는 ChatGPT에서 Codex를 사용할 수 있는 계정이 필요합니다.
  • Codex CLI: 로컬 터미널에서 Codex를 실행하려면 npm i -g @openai/codex로 설치합니다.
  • GitHub 연결: 저장소 코드 수정, 브랜치 생성, 커밋, PR 작업을 맡기려면 GitHub 권한이 필요합니다.
  • Codex 플러그인/커넥터: Slack, Notion, GitHub, Browser Use, Chrome, Documents, Spreadsheets, Presentations 등 필요한 기능을 켭니다.

3. 브라우저 자동화 준비

Playwright

웹사이트를 열고, 클릭하고, 데이터를 읽고, 화면을 검증하는 자동화 도구입니다.

Browser Use (iab)

Codex 안에서 브라우저 상태를 읽고 조작하는 인앱 브라우저 방식입니다.

Codex Chrome Extension

로그인된 Chrome 탭을 Codex가 안전하게 읽고 조작할 수 있게 해주는 런타임입니다.

Chrome 로그인 프로필

Instagram, Google, Naver, Meta처럼 로그인 상태가 중요한 업무는 저장된 Chrome 프로필을 씁니다.

로그인 필요한 사이트는 매번 새 브라우저를 띄우는 방식보다, 본인이 직접 로그인한 Chrome 세션을 Codex가 연결해서 쓰는 방식이 안정적입니다.

4. 데이터베이스와 업무 시스템 권한

Supabase

대시보드, 리포트, 자동화 결과를 저장하고 검증하는 데이터베이스 역할을 합니다.

Slack

자동화 결과, 실패 알림, 일일 리포트를 팀 채널로 보내는 데 필요합니다.

Notion

업무 문서, 운영 지식, 계정 정보 위치, 회의/작업 기록을 참조할 때 사용합니다.

경리나라

월마감, 손익표, 카드·은행 거래 확인, 회계 분류 작업에 필요합니다.

Meta Ads Manager

캠페인, 광고세트, 광고, 예산 구조 분석과 승인된 광고 변경에 필요합니다.

Google Ads / GA4

광고비, 매출, 전환, 유입 데이터를 확인하고 수익 분석에 연결합니다.

Naver SmartStore / SearchAd

네이버 매출, 정산, 검색광고 비용, 상품별 성과를 수집할 때 필요합니다.

Imweb / EZ Admin / KakaoTalk

자사몰 주문, 출고·재고, 발주, 거래처 커뮤니케이션 자동화에 사용합니다.

5. Meta 광고 시스템 구조 이해

  • Campaign: 광고 목적과 큰 예산 단위입니다. 캠페인 예산 방식이면 여기서 예산이 관리됩니다.
  • Ad Set: 타깃, 노출 위치, 일정, 일부 예산이 관리되는 단위입니다. ABO 구조에서는 광고세트 예산이 중요합니다.
  • Ad: 실제 소재, 문구, 랜딩 페이지가 붙는 단위입니다.
  • ASC: 성과가 검증된 소재를 더 넓게 확장할 때 쓰는 자동화 캠페인 구조입니다.
  • 운영 원칙: 분석은 자동화할 수 있지만, 예산 변경·광고 생성·ON/OFF 같은 실제 쓰기 작업은 명확한 승인 후 실행해야 합니다.

6. 현재 활용 중인 자동화 범위

실제 운영에서는 아래 같은 업무를 Codex 에이전트가 정해진 시간이나 요청에 맞춰 수행할 수 있습니다.

수익 대시보드 광고 성과 분석 Meta 세팅 검토 재고 리포트 발주 관리 거래처 카톡 요약 네이버 메일 확인 노무 지원사업 모니터링 보안 점검 Git 업데이트

7. 추천 세팅 순서

  1. Chrome, Git, Node.js, Python 3, Homebrew를 먼저 설치합니다.
  2. Codex 계정에 로그인하고 GitHub 저장소 접근 권한을 연결합니다.
  3. Codex CLI와 Playwright를 설치해 로컬 작업과 브라우저 자동화를 준비합니다.
  4. Slack, Notion, Supabase, GitHub 커넥터를 연결합니다.
  5. Meta, Google, Naver, Imweb, EZ Admin, 경리나라 등 실제 업무 계정은 본인이 직접 로그인해 세션을 준비합니다.
  6. 처음에는 읽기 전용 리포트 자동화부터 시작하고, 광고 변경·DB 쓰기·외부 메시지 발송은 승인 절차를 둡니다.
이 문서는 개인 로컬 경로, 비밀번호, 토큰, 내부 저장소 설치 명령을 포함하지 않는 공유용 요약본입니다.

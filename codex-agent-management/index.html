Codex 에이전트 관리 대시보드

자주 쓰는 Codex 에이전트, 브라우저 방식, 데이터 권한, Slack/Git 운영 규칙을 한 화면에서 고르는 안내서입니다. 복잡한 작업을 시작할 때 먼저 “어떤 에이전트를 불러야 하는지” 빠르게 판단하는 용도입니다.

먼저 관제탑 선택 여러 업무가 섞이면 수익, 마케팅, 지식 라우터 같은 오케스트레이터가 하위 에이전트를 고릅니다.
로그인 작업은 프로필 우선 Instagram, Google, Naver, Meta처럼 세션이 중요한 업무는 저장된 Chrome 프로필이나 Extension을 씁니다.
쓰기 작업은 승인 후 광고 변경, DB 쓰기, Slack/고객 발송, 정산·구매 작업은 명시 승인 후 실행합니다.

1. 브라우저 선택 기준

Browser Use

Codex 안에서 웹페이지를 열고, 클릭하고, 데이터를 읽고, 화면을 검증하는 자동화 방식입니다.

Browser Use (iab)

Codex 인앱 브라우저 상태를 읽고 조작합니다. 사용자 Chrome 탭과 다를 수 있어 최종 공유 URL은 따로 확인합니다.

Playwright

반복 수집, 관리자 화면 조작, 대시보드 렌더링 검증에 씁니다. 로그인 업무는 저장된 Chrome 프로필을 우선합니다.

Codex Chrome Extension

이미 로그인된 Chrome 탭을 Codex가 읽고 조작할 수 있게 해주는 런타임입니다.

로그인된 업무 시스템은 매번 새 브라우저를 띄우는 것보다, 사용자가 직접 로그인한 Chrome 세션을 Codex가 연결해서 쓰는 방식이 안정적입니다. 쿠키나 로그인 정보를 문서에 적거나 복사하지 않습니다.

9. 추천 사용 순서

  1. 업무가 단일 작업인지, 여러 에이전트가 필요한 복합 작업인지 먼저 나눕니다.
  2. 복합 작업이면 수익/마케팅/지식 라우터 중 맞는 관제탑을 먼저 부릅니다.
  3. 브라우저가 필요하면 로그인 여부를 확인하고 Browser Use, Playwright, Chrome Extension 중 하나를 고릅니다.
  4. DB 쓰기, 광고 변경, Slack 발송, 정산·구매처럼 외부 영향이 있는 작업은 승인 단계를 둡니다.
  5. 반복되는 업무는 문서가 아니라 스킬이나 자동화 계약으로 분리합니다.
이 공개 페이지에는 비밀번호, 토큰, 쿠키, 개인 로컬 경로를 넣지 않습니다. 세부 실행 경로와 내부 원본은 로컬 관리 문서에서 관리합니다.
